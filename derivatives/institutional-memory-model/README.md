# Institutional Memory Model (Derivative, Non-Canonical)

This directory provides a non-canonical reference to the **Institutional Memory Model (IMM)** specification.

This content is **non-canonical**.
It introduces no canonical meaning and confers no authority over `canon/`.

---

## What IMM Is

IMM is a **derivative runtime specification** defining admissible data structures and epistemic constraints
for a Brain / SoT organ (e.g., Claim vs Fact vs State, Validity leases, Provenance, Decisions).

It is constrained by the System Momentum Canon and ADR-0 but is not part of the Canon.
