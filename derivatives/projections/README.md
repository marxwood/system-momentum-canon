# Projections — Systemic Derivative Layer

This document describes **Projections** as a derivative layer
built on top of the Brain, without introducing authority,
memory, or normative power.

Projections are **views over Brain state**, never sources of truth.

---

## 1. Derivative Status

Projections are:

- derived from Brain state
- non-authoritative
- non-binding
- non-persistent
- revocable
- regenerable

A Projection cannot exist independently of Brain.

---

## 2. Purpose of Projections

Projections exist to enable:

- perception of system state
- coordination and operation
- analysis and observation

They exist **for agents and systems**, not for the Canon itself.

---

## 3. Canonical Projection Classes

The system recognizes exactly three projection classes:

### 3.1 Public Projections

- external-facing representations
- communicative in nature
- no operational authority

Examples: articles, public feeds, published views.

---

### 3.2 Operational Projections

- coordination-oriented representations
- reflect current operational state
- may contain Operational State Signals (OSS)

Operational Projections must not create commitments.

---

### 3.3 Analytical Projections

- observational and evaluative views
- support analysis and reflection
- no direct action authority

Examples: summaries, metrics, evaluations.

---

## 4. Boundary Rules

Projections must not:

- modify Brain state
- introduce meaning
- assert validity
- create institutional memory
- trigger governance actions

Any system violating these rules is non-canonical.

---

## 5. Closure Rule

The set of projection classes is **closed**.

New projection classes require an explicit Canon amendment.
