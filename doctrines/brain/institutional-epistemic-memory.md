# Institutional / Epistemic Memory Domain — Doctrinal View

This document explains the Institutional / Epistemic Memory Domain (IEM)
from a human and systemic perspective.

It clarifies what it means for a system to **remember institutionally**,
why memory is inseparable from responsibility,
and why most systems fail precisely at this layer.

---

## 1. What Institutional Memory Is

Institutional memory is where the system records
**what it is willing to stand behind**.

It contains:

- claims
- decisions
- states
- validity outcomes

If something enters IEM,
the system becomes accountable for it.

Memory here is not recall.
It is **commitment**.

---

## 2. Memory vs Storage

Institutional memory is often confused with storage.

- Storage holds data.
- Memory holds responsibility.

A log file is not memory.
A database is not memory.

Memory exists only where:

- something was asserted
- someone (or something) is accountable
- consequences follow

---

## 3. What May Enter IEM

Only outputs that passed explicit authorization may enter IEM:

- validated claims
- governed decisions
- declared states

Everything else must remain outside.

If speculative or perceptual data is persisted here,
the system institutionalizes uncertainty as fact.

---

## 4. Time, Irreversibility, and Audit

Institutional memory is temporal.

Once committed:

- it has a timestamp
- it has provenance
- it cannot be silently erased

Corrections are possible,
but only as **new memory entries**,
never by rewriting the past.

This is what makes audit possible.

---

## 5. Relationship to Other Domains

- Perception feeds candidates — never memory
- Cognition proposes — never commits
- Semantic Authority defines meaning — never stores events
- Action / Governance commits — memory begins there
- Reflection inspects memory — but does not alter it

IEM is the anchor point
that stabilizes the entire system.

---

## 6. Responsibility and Accountability

Every memory entry implies:

- responsibility
- potential consequence
- historical weight

A system without institutional memory
cannot learn,
cannot be audited,
and cannot be trusted.

A system with careless memory
cannot forget its mistakes.

---

## 7. Typical Failure Modes

Common failures include:

- storing percepts as facts
- persisting reasoning artifacts
- overwriting history
- erasing uncomfortable decisions
- treating analytics as memory

Most institutional failures
are memory failures.

---

## 8. Practical Examples

Examples of institutional memory:

- a published claim
- a recorded decision
- an official position
- a declared state of affairs

If it can be denied later,
it was never memory.

---

## Closing Note

Institutional memory is where systems
become historical actors.

Without it, systems react.
With it, systems are judged.
