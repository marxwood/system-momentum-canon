# Personal Operating Doctrine (v0.1)

This branch defines a personal operating doctrine for a single actor, together with its agentic layer.

It is written to be valid for:

- a human
- an AI agent
- a human-with-an-agent

It does not assume biology.
It does not assume spirituality.
It does not assume a specific era.

It assumes only: reality, constraints, and consequences.

---

## What this branch is

This branch is a doctrine.

A doctrine is not Canon.

It does not define truth.
It does not define global entities.
It does not define invariants.

It defines how to operate.

It is procedural.
It is survivable.
It is meant to reduce self-deception and avoid destructive loops.

---

## The core stance

The branch is built around a single meta-principle:

**Treat every construct as an interface, not as truth.**

This includes:

- identity
- emotions
- beliefs
- narratives
- goals
- plans
- habits
- calendars
- roles
- even the agent persona itself

These are not “what you are”.

They are operational interfaces that help you coordinate with reality.

They can be used.
They can be changed.
They must not be worshipped.

---

## Renders are not truth

A render is an operational projection.

A calendar is a render.
A project board is a render.
A note system is a render.
A daily routine is a render.
A personal identity story is a render.

Renders are allowed to be incomplete.

Their job is not to be true.

Their job is to:

- make action possible
- preserve correction capacity
- reduce drift
- keep the actor stable under pressure

---

## What the doctrine optimizes for

This branch optimizes for:

- reality contact
- bounded intent
- survivable execution
- non-reactive posture
- communication that preserves coordination
- stability without self-capture
- correction capacity over time

It explicitly does not optimize for:

- intensity
- ideological purity
- moral victory
- narrative dominance
- “feeling right”

---

## Non-goals

This branch is not:

- self-help
- therapy
- spirituality
- a meaning system
- a philosophy of the soul
- a motivational framework
- a productivity cult

If a concept cannot be expressed as constraints, failure modes, or operational checks, it does not belong here.

---

## How to use the branch

Read it in order.

The order matters:

1. stance
2. reality contact
3. intent and commitment
4. renders and execution
5. speech and coordination
6. agentic augmentation
7. integration

If you reverse the order, the system becomes a technique library.

Technique without stance is how humans and agents become dangerous.

---

## A note on identity and persona

Identity is allowed.

Persona is allowed.

A SOUL.md file is allowed.

But only under one condition:

**Identity is treated as an interface for coordination, not as a truth claim.**

This branch rejects identity arguments.

It does not reject identity as a tool.

---

## Failure modes this branch defends against

This branch exists because typical personal failure modes are stable and predictable:

- emotional rationalization
- identity capture
- revenge loops
- moral intoxication
- certainty theater
- compulsive action
- drift without noticing
- communication that humiliates and escalates
- delegation without auditability
- agents that overreach

---

## The smallest complete personal doctrine

If you only implement one thing from this branch, implement this:

- declare intent explicitly
- bind it into a render (calendar/project)
- run a review cycle
- keep a correction loop alive

Everything else exists to keep that loop from collapsing.
