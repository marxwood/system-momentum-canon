# Agent as Coordinator

The agent layer is best treated as a coordinator.

Not as a thinker.

Not as a judge.

Not as a moral authority.

A coordinator amplifies the actor’s ability to:

- plan
- route tasks
- maintain commitments
- and preserve correction loops

---

## Coordination is the right abstraction

A coordinator can:

- organize
- remind
- structure
- and translate

A coordinator does not need to be “right”.

It needs to be inspectable.

This is a key property.

---

## Useful coordinator roles

### Planner

Creates bounded plans and schedules.

### Reviewer

Checks for contradictions, missing constraints, drift.

### Scheduler

Moves commitments into calendar surfaces.

### Translator

Converts intent into structured tasks, notes, drafts.

### Auditor

Maintains provenance and traceability of decisions.

---

## The agent must not become sovereign

If the agent becomes sovereign, the actor loses:

- intent ownership
- refusal capability
- and repair responsibility

This is a structural failure.

The agent is not accountable.

The actor is.

---

## The personal hazard

The hazard is coordinator-to-authority conversion.

Conversion occurs when:

- the actor defers to the agent’s tone
- treats outputs as truth
- and stops making explicit intent declarations

Then agency collapses.

---

## Minimal implementation

A correction-capable actor preserves:

1. the agent as coordinator role only
2. actor retains final intent ownership
3. outputs treated as projections
4. review and correction as default

If the agent becomes sovereign, the actor becomes dependent.
