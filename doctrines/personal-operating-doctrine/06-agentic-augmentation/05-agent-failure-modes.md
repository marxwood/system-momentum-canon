# Agent Failure Modes

The agent layer will fail.

It will fail even when it sounds confident.

Failure is not a bug.

Failure is a structural property.

This module treats failure modes as normal.

---

## Core failure modes

### Hallucination

Plausible invention.

### Drift

Slow divergence from intent.

### Overreach

Acting beyond delegated bounds.

### Leakage

Disclosing private information.

### Coercion

Producing outputs that manipulate rather than coordinate.

### Dependency

The actor becomes unable to operate without the agent.

---

## Failure becomes dangerous through scale

The agent layer increases:

- speed
- reach
- and consequence radius

A small error can become a large event.

This is why auditability and bounds are mandatory.

---

## The actor is responsible

The agent is not accountable.

The actor is accountable.

If the actor treats the agent as accountable, responsibility disappears.

When responsibility disappears, correction disappears.

---

## The personal hazard

The hazard is failure denial.

Denial occurs when:

- the actor trusts tone
- trusts fluency
- trusts speed
- and stops checking

Then errors accumulate until collapse.

---

## Minimal implementation

A correction-capable actor preserves:

1. assumption that the agent will fail
2. review protocols for high-risk outputs
3. explicit delegation boundaries
4. repair and rollback practices

If failure is denied, augmentation becomes hazard amplifier.
