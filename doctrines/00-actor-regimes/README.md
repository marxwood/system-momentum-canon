It exists to prevent silent drift about “who the actor is” and what “accountability / legitimacy / repair” mean.

The three main branches (personal / organizational / civilizational) are written under a default regime to stay:

- operational
- non-speculative
- and consequence-grounded

Unless a file explicitly states otherwise, interpret doctrine content under **Regime A (Human-final)**.

---

## Why this matters

Many systems treat “actor” as “anything that can act”.

This doctrine uses a stricter meaning:

An actor is an entity that can be a **final bearer** of:

- responsibility
- enforceable constraints
- repair obligations
- and legitimacy boundaries

Agents can execute actions without being final bearers.

That difference changes everything.

---

## Regimes

- Regime A — Human-final (default)
- Regime B — Hybrid (bounded autonomous agents)
- Regime C — Agent-final (fully agentic actors)

See the regime files for definitions and reading rules.

---

## Core safety rule

Speculation is not the problem.

Speculation **without mechanics** is the problem.

A regime is “operational” only when it specifies:

- legitimacy substrate (who grants permission, who can revoke)
- enforcement (how constraints are applied)
- coercion primitives (who can block resources / actions, under what rules)

Without these, “civilization” becomes metaphor.
