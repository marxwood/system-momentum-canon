# Roles and Delegation

Organizations scale by delegation.

Delegation is not optional.

If delegation is informal, it becomes unbounded.

If delegation is unbounded, it becomes capture.

This section defines roles and delegation as coordination interfaces.

---

## Roles are interfaces

A role is not a title.

A role is an interface that defines:

- what actions are permitted
- what evidence is required
- what constraints apply
- and who owns repair

Titles create status.

Interfaces create accountability.

---

## Delegation is not trust

Delegation is not a moral statement.

Delegation is a capability transfer.

Capability transfers must be bounded.

Otherwise the organization becomes:

- a collection of rogue actors
- or a collection of fearful non-actors

Both outcomes destroy correction capacity.

---

## The organizational hazard

The hazard is delegation becoming:

- implicit
- social
- and status-driven

Status-driven delegation produces:

- immunity
- blame diffusion
- and credibility theater

This is how organizations become coordinated but not corrigible.

---

## What this section covers

This section defines:

- boundaries between human and system roles
- a role taxonomy usable as responsibility surfaces
- ownership and accountability bindings
- lifecycle discipline for short-lived vs long-lived execution units
- delegation contracts as constraint carriers
- escalation, override, and kill switches

---

## Minimal implementation

A correction-capable organization must preserve:

1. explicit delegation boundaries
2. ownership bindings for delegated work
3. escalation paths that are survivable
4. override mechanisms that are real, not symbolic

If delegation cannot be bounded, capability becomes capture.
