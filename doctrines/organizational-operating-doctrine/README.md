# Organizational Operating Doctrine (v0.1)

This branch defines an organizational operating doctrine.

It is written for organizations that operate as agentic systems, meaning:

- work is executed through a mix of humans and agents
- actions have real consequences
- coordination depends on shared reality and bounded authority
- drift and capture are normal failure modes

This branch does not assume a specific industry.
It does not assume a specific technology stack.
It does not assume a specific era.

It assumes only: incentives, constraints, power, and consequences.

---

## What this branch is

This branch is a doctrine.

A doctrine is not Canon.

It does not define truth.
It does not define global entities.
It does not define invariants.

It defines how organizations operate.

It is procedural.
It is survivable.
It is meant to preserve correction capacity under scale, incentives, and pressure.

---

## The core stance

The branch is built around a single meta-principle:

**Treat every organizational construct as an interface, not as truth.**

This includes:

- org charts
- roles and titles
- processes and rituals
- OKRs and metrics
- policies
- incident procedures
- evaluation gates
- performance systems
- “culture”
- the organization’s narrative about itself
- the agent personas the org deploys

These are not “what the organization is”.

They are operational interfaces for coordination.

They can be changed.
They must be audited.
They must not be worshipped.

---

## Why this matters in organizations

At organizational scale, interface-worship is the default.

It appears as:

- bureaucracy treated as legitimacy
- metrics treated as reality
- titles treated as authority
- process treated as safety
- culture slogans treated as ethics
- dashboards treated as truth
- compliance treated as correctness

This branch exists to prevent the organization from becoming a machine that only protects its own interfaces.

---

## What the doctrine optimizes for

This branch optimizes for:

- correction-capable execution
- bounded autonomy
- auditability of action
- provenance of decisions
- survivable delegation
- governance without paralysis
- incentives that do not destroy truth
- security under adversarial reality

It explicitly does not optimize for:

- narrative cohesion at all costs
- “alignment theater”
- productivity slogans
- AI hype
- speed without accountability
- control without correction

---

## Non-goals

This branch is not:

- a vendor playbook
- a technology architecture guide
- an HR manifesto
- a motivational framework
- a culture essay collection

If a concept cannot be expressed as constraints, failure modes, or operational checks, it does not belong here.

---

## How to use the branch

Read it as an operating model, not as a library.

The order matters:

1. roles and boundaries
2. intent and execution loops
3. memory, provenance, and correction
4. governance and control
5. culture and incentives
6. operating model architecture
7. economics of transformation
8. adversarial reality
9. scaling coordination
10. external interfaces

The goal is not “AI adoption”.

The goal is a system that can act without losing correction capacity.

---

## A note on culture

Culture is allowed.

But culture is not truth.

Culture is an interface that shapes behavior through:

- incentives
- fear gradients
- status games
- permission structures
- punishment and reward

This branch treats culture as a controllable surface, not as identity.

---

## Failure modes this branch defends against

This branch exists because organizational failure modes are stable:

- capture by incentives
- capture by status games
- capture by process
- capture by metrics (Goodhart pressure)
- tool sprawl and truth fragmentation
- delegation without accountability
- agents operating outside bounded authority
- hallucination-driven execution
- compliance theater
- security collapse under adversarial pressure

---

## The smallest complete organizational doctrine

If you only implement one thing from this branch, implement this:

- explicit intent declaration for actions
- bounded authority for agents and humans
- auditability of execution
- a correction loop that can stop and reverse harmful drift

Everything else exists to keep that loop alive under scale.
