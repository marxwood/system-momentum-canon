# Source of Truth

Organizations collapse when they have multiple incompatible sources of truth.

They call it “alignment issues”.

In reality, it is epistemic fragmentation.

Fragmentation makes correction impossible.

---

## Source of truth is not a database

Source of truth is not a tool.

Source of truth is an authority boundary.

It defines:

- which claims are authoritative
- which claims are provisional
- and how disputes are resolved

Tools can store truth.

Tools cannot define it.

---

## Truth boundaries are interfaces

A source of truth is an interface.

It must be able to answer:

- what is currently treated as true
- why it is treated as true
- what would falsify it
- and who owns repair if it is wrong

Without these, the “source of truth” is only a storage location.

---

## The organizational hazard

The hazard is truth drift.

Truth drift occurs when:

- local teams fork reality
- metrics become ideology
- and decisions are made from incompatible assumptions

Truth drift produces:

- duplicated work
- blame diffusion
- and silent capture

Because capture thrives in ambiguity.

---

## Failure modes

### Multiple authoritative realities

Two systems claim authority.

Disputes become political.

### Truth by seniority

Truth is decided by status.

This becomes capture.

### Truth by metrics

Metrics become unquestionable.

This becomes ideology.

### Truth by narrative

Stories substitute for evidence.

This becomes theater.

---

## Minimal implementation

A correction-capable organization must preserve:

1. explicit authority boundaries for truth claims
2. dispute resolution paths
3. falsifiability conditions for key claims
4. repair ownership when truth is wrong

If truth boundaries are not explicit, the organization will coordinate through power.
