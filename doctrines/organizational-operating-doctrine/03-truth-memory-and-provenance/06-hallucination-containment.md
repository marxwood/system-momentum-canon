# Hallucination Containment

Organizations fail when error becomes persuasive.

This is not new.

What is new is the speed at which plausible error can be produced.

This module treats hallucination as a general class of failure:

- confident false claims
- provenance loss
- and unverifiable outputs

The term is used as shorthand.

The mechanism is the real subject.

---

## Containment is not banning

Containment is not prohibition.

Containment is the discipline of:

- bounding where unverifiable outputs may be used
- enforcing provenance for high-impact claims
- and requiring contradiction capacity

A correction-capable organization assumes:

- some outputs will be wrong
- and persuasion will be weaponized

---

## The organizational hazard

The hazard is persuasive error becoming legitimacy.

This occurs when:

- outputs are accepted without inspection
- provenance is absent
- and contradiction is socially unsafe

When persuasive error becomes legitimacy, correction collapses.

---

## Failure modes

### Output-as-truth

Outputs are treated as authoritative.

This produces silent drift.

### Provenance stripping

Outputs are copied without sources.

Audit becomes impossible.

### Social intimidation

People stop contradicting persuasive claims.

Truth becomes status.

### Automation laundering

Error is accepted because “the system said so”.

This becomes delegated delusion.

---

## Minimal implementation

A correction-capable organization must preserve:

1. bounded zones where unverifiable outputs are allowed
2. provenance requirements for high-impact claims
3. contradiction as a normal function
4. repair ownership when persuasive error causes harm

If persuasive error becomes legitimacy, the organization becomes capturable through outputs.
