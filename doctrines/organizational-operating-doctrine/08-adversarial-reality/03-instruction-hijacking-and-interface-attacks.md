# Instruction Hijacking and Interface Attacks

Organizations are attacked through interfaces.

Interfaces are not only technical.

Interfaces include:

- prompts
- tickets
- emails
- chats
- and documents

Instruction hijacking is the general mechanism of:

- forcing execution to follow hostile intent

This module treats interface attacks as a normal risk.

---

## Hijacking is a legitimacy problem

Hijacking succeeds when:

- intent is not declared
- constraints are not enforced
- and provenance is not checked

The attack is not magic.

The attack is governance failure.

---

## The organizational hazard

The hazard is execution that trusts input.

Trusting input is the default.

Default trust is exploitability.

---

## Failure modes

### Context poisoning

Inputs embed hidden intent.

Execution follows it.

### Authority spoofing

Attackers impersonate legitimacy.

Work is executed under false authority.

### UI social engineering

People are manipulated through interfaces.

This bypasses technical controls.

### Constraint bypass under urgency

Urgency is used to erase checks.

This accelerates harm.

---

## Minimal implementation

A correction-capable organization must preserve:

1. intent declaration and validation for high-impact execution
2. provenance checks for instruction sources
3. constraints that cannot be erased by input
4. escalation paths when input is suspicious

If interfaces are trusted by default, the organization will be controlled through them.
