# Organizational Operating Doctrine

This branch describes how organizations operate as coordinated agency.

It is written for environments where:

- work is decomposed and delegated
- authority is layered
- incentives distort behavior
- and coordination must survive scale

This is not a transformation manifesto.

It is an operating doctrine for correction-capable organizational order.

---

## What this branch is

This branch treats an organization as:

- coordinated agency
- expressed through roles
- executed through pipelines
- bounded through constraints
- stabilized through accountability
- and shaped through incentives

The organization is not a “culture”.

Culture is a downstream effect.

The organization is not “a family”.

Families do not scale.

---

## The core stance

The stance of this branch is:

**Organizations must preserve correction capacity while executing.**

Execution without correction becomes:

- capture
- delusion
- and unbounded power

Correction without execution becomes:

- paralysis
- and institutional theater

The goal is not harmony.

The goal is survivable, inspectable coordination.

---

## Constructs as interfaces

This branch treats organizational constructs as interfaces.

Not as sacred objects.

Not as slogans.

Examples:

- a role is an interface
- a delegation is an interface
- an approval is an interface
- an incident is an interface
- a policy is an interface
- a ledger is an interface

If the interface cannot be inspected, it cannot preserve correction.

---

## The organizational hazard

The hazard at organizational scale is not disagreement.

It is capture.

Capture inside organizations appears as:

- immunity of executives or factions
- metrics that cannot be contradicted
- dependencies that cannot be exited
- audits that exist as theater
- and learning loops that become PR

Capture produces order.

It destroys correction.

---

## What this branch covers

This branch covers:

- roles, delegation, escalation, override
- intent-to-execution pipelines
- truth, provenance, institutional memory
- governance, risk, evaluation, incident response
- incentives, status dynamics, credibility theater
- operating model design for complex human+system work
- economics, dependence, lock-in, labor redesign
- adversarial reality: abuse, leakage, injection, sabotage
- scaling coordination without bureaucracy
- external interfaces: regulation, reputation, markets

---

## Non-goals

This branch is not:

- a vendor playbook
- a tool catalog
- a software architecture spec
- a culture manifesto
- a productivity ideology
- a moral system

It does not prescribe “best practices”.

It defines failure modes and minimal survivable conditions.

---

## Minimal implementation

An organization that wants correction-capable order must preserve:

1. intent before action
2. bounded delegation
3. accountability ledgers
4. survivable contradiction
5. incentives that do not reward capture
6. exit paths from dependencies

If these do not exist, capability amplifies capture.
