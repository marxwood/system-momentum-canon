# Scope and Non-Goals

This branch defines how organizations maintain correction-capable order while executing coordinated work.

It assumes:

- delegation is unavoidable
- authority is layered
- incentives distort behavior
- and failure is normal

The purpose is not to make organizations “good”.

The purpose is to make organizations corrigible.

---

## Scope

This branch covers organizational mechanics that determine whether correction survives:

- role boundaries and delegation limits
- execution pipelines and interruption handling
- provenance, auditability, and institutional memory
- governance, risk classes, and evaluation gates
- incentive design and status dynamics
- dependence, lock-in, and exit paths
- adversarial reality: abuse, leakage, sabotage
- scaling coordination without bureaucratic paralysis
- external interfaces and legitimacy pressures

It treats these as operational interfaces, not as narratives.

---

## Non-goals

This branch is not:

- a transformation program
- a leadership philosophy
- a motivation system
- a vendor adoption playbook
- a blueprint for organizational charts
- a software architecture specification
- a compliance manual
- a manifesto about the future of work

It does not attempt to define:

- what “good culture” is
- what a “healthy company” looks like
- what ideology should guide the organization

---

## What this branch refuses

This branch refuses common shortcuts that produce capture:

- “trust us” governance
- hero-centric execution
- metrics-as-truth
- audit-as-theater
- culture-as-substitute-for-structure
- punishment-as-learning
- loyalty-as-legitimacy

These shortcuts can produce speed.

They also destroy correction capacity.

---

## Minimal stance

Use this branch when the question is:

- does this mechanism preserve correction under pressure?
- does this interface prevent capture?
- does this reduce reliance on coercion inside the organization?

If the answer is “no”, the mechanism may still work.

It will work by amplifying capture.
