# Capability Boundaries

Organizations collapse when capability is unbounded.

Unbounded capability is power without constraints.

Power without constraints becomes capture.

This module treats capability boundaries as the organizational equivalent of:

- “what is allowed”
- and “what is impossible by design”

---

## Boundaries are not policies

Boundaries are not statements.

Boundaries are enforced constraints.

A boundary that can be bypassed socially is not a boundary.

It is a suggestion.

---

## The organizational hazard

The hazard is boundary erosion.

Boundary erosion occurs when:

- exceptions become normal
- urgency bypasses constraints
- and high-status actors ignore boundaries

Boundary erosion produces:

- immunity
- and silent drift

---

## Failure modes

### Exception culture

Boundaries exist, but exceptions are routine.

### Status bypass

Leaders bypass constraints.

This is capture.

### Boundary confusion

No one knows what is allowed.

People act by fear or rumor.

### Unowned boundaries

No one owns boundary repair.

Boundaries decay.

---

## Minimal implementation

A correction-capable organization must preserve:

1. explicit capability boundaries for high-impact actions
2. symmetric enforcement (no status bypass)
3. expiration for exceptions
4. repair ownership for boundary violations

If boundaries cannot be enforced, governance becomes theater.
