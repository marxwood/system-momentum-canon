# Work Queues and Task Pools

Organizations coordinate through queues.

Even when they pretend they do not.

Work appears as:

- requests
- incidents
- opportunities
- and obligations

If work is not routed through explicit queues, it routes through:

- status
- relationships
- and fear

That routing produces capture.

---

## Task pools are not efficiency hacks

Task pools are not about productivity.

They are about corrigibility.

A pool makes it possible to:

- see what work exists
- see who owns it
- see what is blocked
- and see what is being avoided

Without pools, work becomes invisible.

Invisible work becomes unaccountable.

---

## The organizational hazard

The hazard is shadow routing.

Shadow routing occurs when:

- leaders assign work privately
- urgent work bypasses queues
- or factions control what is visible

Shadow routing produces:

- immunity
- blame diffusion
- and execution theater

---

## Failure modes

### Queue bypass

High-impact work is executed outside the pool.

This destroys auditability.

### Queue capture

A faction controls what enters the pool.

This is internal capture.

### Pool without ownership

Tasks exist, but owners do not.

This becomes blame diffusion.

### Pool without evaluation gates

Tasks flow, but nothing is checked.

This produces drift.

---

## Minimal implementation

A correction-capable organization must preserve:

1. explicit work queues for high-impact work
2. ownership assignment at intake
3. evaluation gates attached to tasks
4. visibility into queue bypass events

If queues are not explicit, the organization will route work through status.

Status routing produces capture.
