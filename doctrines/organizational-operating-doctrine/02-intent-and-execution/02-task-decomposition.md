# Task Decomposition

Organizations fail when intent is decomposed into tasks that are:

- unbounded
- unowned
- or unevaluable

Task decomposition is the discipline of turning intent into bounded work.

Bounded work is what makes correction possible.

---

## Decomposition is not planning theater

Decomposition is not documentation for appearance.

Decomposition is the creation of:

- clear action surfaces
- clear evaluation gates
- and clear repair ownership

If these are not produced, decomposition is theater.

---

## The organizational hazard

The hazard is decomposition drift.

Decomposition drifts into:

- vague tickets
- unbounded mandates
- and hero tasks

Hero tasks produce:

- blame diffusion
- urgency exemptions
- and silent capture

---

## Failure modes

### Mandate tasks

Tasks are written as goals.

Goals cannot be evaluated.

### Infinite scope

Tasks expand during execution.

Scope creep becomes normal.

### Unowned tasks

No one owns repair.

Failures repeat.

### Unevaluable tasks

Success cannot be measured without narrative.

Narrative becomes legitimacy.

---

## Minimal implementation

A correction-capable organization must preserve:

1. bounded tasks with explicit scope
2. evaluation criteria before execution
3. repair ownership bindings
4. escalation triggers when tasks expand

If tasks cannot be bounded, execution becomes heroism.
